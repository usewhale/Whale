## Codebase Patterns
- Cross-platform shell behavior should live in `internal/shell` with injectable `GOOS` and `LookPath` dependencies so tests do not depend on the host platform.
- Windows shell resolution should prefer `pwsh`, then `powershell.exe`, and should not fall back to `cmd.exe`, Git Bash, MSYS Bash, `bash`, or `sh`.
- Foreground `exec_shell` should resolve the shell before creating `exec.Cmd`; resolver failures are returned as `shell_unavailable` tool errors without starting a process.
- Background `exec_shell` tasks should resolve the shell before creating `exec.Cmd`; resolver failures are recorded on the task as `failed` with stderr set to the resolver error and nil exit code.
- Hooks should resolve the shell before creating `exec.Cmd`; resolver failures should return `SpawnErr` so blocking hook events preserve existing block semantics.
